# 스킬 관리

[← 목차로 돌아가기](../README.md) · [이전: MCP 서버](08-mcp.md)

---

스킬(Skill)은 반복적인 워크플로우를 재사용 가능한 명령어로 패키징한 것입니다. 예를 들어 "코드 리뷰" 과정을 스킬로 만들면, 매번 같은 지시를 반복하지 않아도 됩니다. 토큰 절약을 위해 3단계(Tier)로 나눠서 필요한 만큼만 로드합니다.

## 3-Tier 로딩 시스템

| Tier | 로드 시점 | 내용 |
|------|-----------|------|
| **Tier 1** | 항상 | 메타데이터: `name` (64자), `description` (1024자) |
| **Tier 2** | 관련 시 | SKILL.md 마크다운 본문 (<5k 토큰) |
| **Tier 3** | 필요 시 | scripts/, references/, assets/ |

## 스킬 디렉토리 구조

```
~/.claude/skills/              # 글로벌 스킬
└── code-review/
    ├── SKILL.md               # 필수: 메타데이터 + 지침
    ├── scripts/               # 선택: 실행 가능한 코드
    ├── references/            # 선택: 참조 문서
    └── assets/                # 선택: 템플릿, 아이콘

{project}/.claude/skills/      # 프로젝트 스킬
```

## SKILL.md 구조

```markdown
---
name: code-review
description: 코드 리뷰 수행 시 사용. PR, 코드 변경, 품질 검토 요청 시 자동 활성화.
---

# 코드 리뷰 지침

## 검토 영역
1. **가독성**: 변수명, 함수명, 주석
2. **보안**: SQL 인젝션, XSS, 인증/인가
3. **성능**: 알고리즘 복잡도, 메모리 사용
4. **테스트**: 테스트 커버리지, 엣지 케이스

## 출력 형식
- 긍정적 측면 먼저 언급
- 개선 사항은 구체적 제안과 함께
- 우선순위: Critical / High / Medium / Low
```

## 중요 사항

- **name**: 64자 이하
- **description**: 1024자 이하, **트리거 조건을 반드시 명시** (가장 중요!)
- **지침**: 500줄 이하 권장 (공식 문서 기준)
- **description에 "언제 사용할지" 포함** — 본문이 아닌 메타데이터에!

## 스킬 vs CLAUDE.md vs Hooks

| | CLAUDE.md | 스킬 | Hooks |
|-|-----------|------|-------|
| **성격** | 항상 활성 | 필요 시 활성 | 이벤트 트리거 |
| **용도** | 프로젝트 컨텍스트 | 워크플로우 | 자동화 |
| **실행** | 수동 (Claude 판단) | 수동/자동 | 결정론적 |
| **예시** | 기술 스택 명시 | 코드 리뷰 프로세스 | 자동 포맷팅 |
