# 테스팅 전략

## 원칙
- 로직 변경 시 테스트 필수
- 테스트는 행위를 검증 (구현이 아님)
- Given-When-Then 패턴 사용
- 테스트 이름은 기대 행위를 설명

## 구조
- 단위 테스트: `__tests__/` 또는 `*.test.ts`
- 통합 테스트: `tests/integration/`
- E2E 테스트: `tests/e2e/`

## 커버리지
- 새 기능: 최소 80% 커버리지
- 버그 수정: 반드시 회귀 테스트 추가
- 유틸리티 함수: 100% 커버리지 목표

## 모킹
- 외부 의존성만 모킹 (DB, API, 파일시스템)
- 내부 모듈 모킹 최소화
- 모킹보다 의존성 주입 선호

## 명령어
- `npm test` — 전체 테스트
- `npm test -- --watch` — 변경 감지 모드
- `npm test -- --coverage` — 커버리지 리포트
