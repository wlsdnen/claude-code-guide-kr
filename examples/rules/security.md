# 보안 규칙

## 민감 데이터
- .env 파일 절대 커밋 금지
- API 키, 시크릿은 환경 변수로만 관리
- 로그에 민감 정보 출력 금지
- 하드코딩된 자격증명 금지

## 입력 검증
- 모든 사용자 입력은 서버 사이드에서 검증
- Zod/Joi 등 스키마 검증 라이브러리 사용
- SQL 쿼리에 파라미터 바인딩 필수 (ORM 사용 권장)
- HTML 출력 시 XSS 방지 이스케이핑

## 인증/인가
- JWT 토큰은 httpOnly 쿠키에 저장
- 세션 만료 시간 설정 필수
- API 엔드포인트별 권한 검사
- CORS 설정 명시적으로 관리

## 의존성
- 정기적으로 `npm audit` 실행
- 알려진 취약점이 있는 패키지 즉시 업데이트
- 최소 권한 원칙 적용

## 금지 패턴
- `eval()` 사용 금지
- `dangerouslySetInnerHTML` 사용 시 반드시 새니타이징
- `*` CORS 허용 금지 (프로덕션)
